Please use the page structure page to properly understand the function of the page. The code above is for explanation purposes only! 

 

1 - The all elements tag houses all the elements in an overflow hidden box which allows the navigation to be swiped without enterifiering with the X axis. It's important to keep that class.

2- The sidebar shorcuts house all the top right corner icons. To create more or other icons search for the shortcut-call class in the style.css file and use the example there to create an icon. Icons can be found in the images/icons/ folder. The search icon can be removed completely by creating a new sidebar button with a different fuction

3- The search box is dependent on the search button. If you decide to remvoe the search box, you must also remove the search button 

4. The sidebar logo,and the regular logo ( 13 - ) must be given width parameters both in the html and the style.css file 

5. The navigation breadcrumb is the darker bar that includes the sidebar heading 

6. A regular navigation item with no dropdown.

7. A dropdown item. To create a dropdown item copy and paste lines 31 to 38. Add more items in the nav-submenu class and make sure your navigation name ( example Gallery<em></em> ) has no class in the ems. This will make the selection icon a - instead of a plus. To make the dropdown menu shown directly open, add the active-submenu class to your nav-submenu 

8. If the em here has the class dropdown-item it will show a + if the em is completly removed it will show the - icon 

9 .The active-submenu class will set the submenu as opened. 

10 selected-item shows a green ball to signify the menu is active, the unselected-item will show a grey ball. 

11. To create a new page update copy and paste lines 49 to 58 . Be sure to use the sidebar-decoration class if you see a lacking decoration line between elements. 

12. The page content is the class that hosues the elements that will be moved when swiping the page. 

13. The page header houses the toggle icon that deploys the menu the logo (see sidebar logo info on changing ) and the contact icon. this icon can be replaced by locating deploy-contact in the style.css file and replacing it with whatever you require. 

14. The page header clear class is required to create an empty space between the fixed header and the upcoming content. If more space is required add a <div class="decoration"></div> right bellow the header-clear. 

15. The content class and container. The container will house all the elemnets found in the second docuemntation. Pelase check it for more informatin. 

 

Using Wide Gallery/Folio

The wide gallery and the wide folio are two different things. One opens a fullscreen gallery, the other acts like a toggle.

<div class="wide-item-wrapper">
    <div class="wide-item">
        <div class="wide-item-titles">
            <h4>Tile image gallery</h4>
            <p>Add a caption, tap the image, and see the result!</p>
        </div>
        <div class="wide-image">
            <div class="overlay"></div>
            <a class="swipebox-wide" href="images/general-nature/1.jpg" title="Your caption here!"><img class="responsive-image" src="images/general-nature/1w.jpg" alt="img"></a>
        </div>
    </div>
</div>
The following structure will create a one item wide gallery. To create more items copy and paste the wide-item class.
 
The wide item class has the big image found in the wide-image class. This image will be connnected to another image that will be the image showed when this is tapped. The title of the href will be the caption.
The title of the image that will be tapped is the h4 and p found in wide-item-titles. 
 
The wide portfolio uses the same principle with a few modifications
 
<div class="wide-item-wrapper">
    <div class="wide-item">
        <div class="wide-item-titles">
            <h4>One Column</h4>
            <p>Add a caption, tap the image, and see the result!</p>
        </div>
        <div class="wide-image">
            <div class="overlay"></div>
                <a class="tile-wide" href="#"><img class="responsive-image" src="images/general-nature/6w.jpg" alt="img"></a>
         </div>
         <div class="wide-active">
             <div class="content">

             </div>
         </div>
     </div>
</div>

The same principle as above applies. To create a new item, simply copy and paste the wide-item. The titles remain the same, but the noticieable difference is taht now, instead of a image gallery, there is a wide-active class. This houses a content that can hosue containers that can have whatever you want in them. When tapping the big image it will act like a toggle, showing anything within the wide-active class of the tapped image.
